# lab04

Лабораторная работа Лисейчиковой Екатерины

## Отчет по лабораторной работе IV

### Тема

Изучение систем непрерывной интеграции на примере Travis CI.

### Цель работы

Цель работы - добавить CI-конфигурации для Linux и Windows, чтобы проект автоматически собирался на разных платформах.

### Используемое окружение

- основной целевой сценарий: Linux;
- язык: C++11;
- система сборки: CMake;
- дополнительные инструменты добавляются по теме конкретной лабораторной;
- персональные токены и временные адреса не сохраняются в репозитории.

### Структура проекта

- `formatter_lib` - базовая статическая библиотека форматирования;
- `formatter_ex_lib` - расширенная библиотека, использующая `formatter`;
- `solver_lib` - библиотека для решения квадратного уравнения;
- `hello_world_application` - демонстрационное приложение;
- `solver_application` - приложение для вывода корней уравнения;
- `tests` - модульные тесты, начиная с lab05;
- `docs` и `scripts` - дополнительные материалы поздних лабораторных.

### Ход выполнения

- сохранена структура CMake-проекта из предыдущей лабораторной;
- добавлен `.travis.yml` для Linux-сборки с компиляторами gcc и clang;
- добавлен `.appveyor.yml` для Windows-сборки через Visual Studio 2022;
- в CI включены конфигурация, сборка, тестовый target и установка;
- README оформлен как отчет с командами проверки.

### Команды для проверки

```bash
cmake -S . -B _build -DCMAKE_BUILD_TYPE=Release -DBUILD_TESTS=ON
cmake --build _build
ctest --test-dir _build --output-on-failure
cmake --install _build --prefix _install
```

### Домашнее задание

Домашнее задание выполнено внутри этого же репозитория. Подробное описание вынесено в `HOMEWORK.md`.

Кратко по выполнению:

- для Linux используется матрица компиляторов `gcc` и `clang` в `.travis.yml`; 
- для Windows используется `.appveyor.yml` с генератором `Visual Studio 17 2022`; 
- оба сценария запускают CMake одинаковым способом, поэтому поведение сборки остается переносимым;
- конфигурации не содержат персональных токенов или локальных путей.

### Результат

Проект подготовлен к автоматической проверке на Linux и Windows, а домашнее задание по CI оформлено внутри репозитория.
